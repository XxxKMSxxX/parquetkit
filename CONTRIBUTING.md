# Contributing

ParquetKit is **open source, but not open contribution** — in the same spirit
as SQLite and Litestream. The source is MIT-licensed so you can audit the
privacy claims, learn from it and fork it freely. The project itself, however,
is designed to be built and maintained by one person: every line merged here
ships straight to [parquetkit.com](https://parquetkit.com), so review
bandwidth — not code — is the bottleneck.

## What is welcome

- **Bug reports** — please
  [open an issue](https://github.com/XxxKMSxxX/parquetkit/issues). A sample
  file that reproduces the problem (or its schema) helps enormously.
- **Questions and ideas** —
  [GitHub Discussions](https://github.com/XxxKMSxxX/parquetkit/discussions).
- **Security reports** — see [SECURITY.md](./SECURITY.md) for private
  disclosure. Please do not open public issues for vulnerabilities.

## Pull requests

Pull requests are **generally not accepted**. The exception is small, obvious
fixes: typos, broken links, clearly wrong documentation. For anything larger,
please open an issue or discussion first — an unsolicited feature PR will be
closed with a pointer to this document, which is a bad time for everyone.
